**프로젝트 초기 설정**
- Vite + React 프로젝트 구조 설정
- package.json, vite.config.js 구성

**핵심 게임 로직**

- `masking.js`: 형태소 분석 기반 마스킹 및 매칭 로직
- `storage.js`: 로컬 스토리지 기반 통계 관리

**UI/UX 컴포넌트**
- `Header.jsx`: 헤더 (로고, 통계/도움말 버튼)
- `GameBoard.jsx`: 마스킹된 문서 표시, 진행률 바
- `InputBar.jsx`: 하단 고정 입력창, Cold/Warm/Hot 피드백 애니메이션
- `ResultScreen.jsx`: 게임 완료 결과 및 공유 기능
- `StatsModal.jsx`: 사용자 통계 모달
- `HelpModal.jsx`: 게임 방법 설명 모달

**반응형 디자인**
- 모바일 우선 디자인
- PC: 3컬럼 레이아웃 (광고-게임-광고)
- 모바일: 전체 너비 게임 영역 + 하단 광고

## 🔧 기술적 결정

### 1. 형태소 분석 대체
**문제**: Mecab-ko 서버리스 포팅이 복잡  
**해결**: 사전 가공된 토큰 데이터 사용 (MVP 단계)  
**향후**: 서버 사이드 형태소 분석 API 연동

### 2. 데이터 소스
**결정**: 위키백과 → 나무위키 (사용자 피드백 반영)  
**이유**: 한국어 콘텐츠 풍부, 친숙한 문체

### 3. 상태 관리
**결정**: React Hooks + LocalStorage  
**이유**: 외부 라이브러리 없이 MVP 구현 가능
